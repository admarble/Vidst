Overview

The Twelve Labs model implementation for video understanding and analysis using the Twelve Labs API.

Resource Management

The model implements robust resource management to ensure proper cleanup of system resources, API connections, and temporary files. This is particularly important for long-running applications and systems processing multiple videos.

Cleanup Mechanisms

The model provides two main cleanup mechanisms:

  1. Explicit Cleanup (Recommended)

    model = TwelveLabsModel(api_key)
try:
      await model.process(video_data)
finally:
      await model.close()  # Explicit cleanup

  2. Automatic Cleanup (Fallback)

    The model implements __del__ for automatic cleanup during garbage collection, but this should not be relied upon as the primary cleanup mechanism.

    Warning

    Do not rely on automatic cleanup in production code. Always use explicit cleanup via close() when possible.

Cleanup Scenarios

The cleanup system handles various scenarios:

  1. Normal Operation

    • Explicit close() call

    • All resources properly released

    • Active tasks cancelled

    • API client closed

  2. Running Event Loop

    • Non-blocking cleanup

    • Task scheduled in existing loop

    • Asynchronous resource release

  3. No Event Loop

    • Temporary loop creation

    • Synchronous cleanup

    • Loop properly closed

  4. Interpreter Shutdown

    • Graceful cleanup skip

    • No errors raised

    • Safe termination

Best Practices

Follow these guidelines for proper resource management:

  1. Always use explicit cleanup:

    async with TwelveLabsModel(api_key) as model:
      await model.process(video_data)

  2. Handle cleanup in error cases:

    try:
      await model.process(video_data)
except Exception:
      # Handle error
      raise
finally:
      await model.close()

  3. Monitor active tasks:

    # Check active tasks before cleanup
if model._active_tasks:
      logger.warning(f"Cleaning up {len(model._active_tasks)} active tasks")
await model.close()
Implementation Details

The cleanup implementation is designed to be:

  • Robust: Handles all error cases gracefully

  • Non-blocking: Uses asynchronous cleanup when possible

  • Complete: Ensures all resources are released

  • Safe: Prevents resource leaks and shutdown errors

The cleanup system is thoroughly tested for various scenarios:

# Test explicit cleanup
await model.close()
assert len(model._active_tasks) == 0
